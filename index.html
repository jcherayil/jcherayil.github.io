PixelDust

Supporting dynamic area of interest tagging in programming studies with eye tracking

Eye tracking studies are valuable for evaluating programming environments, but annotating what the programmer is looking at in a dynamic environment can be repetitive, timeconsuming, and error prone. Through a participatory design exercise with two eye tracking researchers, I identified three significant challenges: search, extraction of code, and annotating transient objects. By applying computer vision algorithms to video traces, I developed a mixed-initiative system called PixelDust, which allows the researcher to train a system to recognize different objects on a screen. My preliminary results demonstrate the versatility of the approach; for example, the system can recognize return statements, method signatures, tool tips, and dialog boxes.

Biometric Authentication

Discovering the optimal login flow for Deloitte employees using biometrics as a form of multi-factor authentication

Until this project, Deloitte apps did not share a common strategy for multi-factor authentication. Striving for consistency in our suite of apps, I set out to find the optimal login experience for users in terms of security, design, and user experience. First, I did some research on existing APIs that enable biometric authentication, in order to find out what kinds of login scenarios were possible. I enumerated several such scenarios, each varying in certain aspects, such as fallback mechanism in case of biometric failure. Then, I created demo apps to showcase each scenario, and received user feedback from teams that focus on design, security, and user experience. After modifications, I was able to recommend a seamless login experience for users that was sharp-looking, secure, and user-friendly. This login experience is now standard for all of our mobile apps.

Pre-Populated Expense Reports from Receipts

Leveraging OCR technology to pull information from receipts and pre-populate fields in an itemized expense report.

Filling out expense reports in Deloitte's existing expensing app is particularly time-consuming for hotel receipts, in which every charge (e.g. room charge, food expenses, tax) must be itemized per day. Several employees report spending hours expensing hotel receipts even after just a few days of travel. In this demo, we demonstrated the power of using optical character recognition (OCR) to extract all charges from a PDF receipt. Once extracted, our program sorted the charges by date and filled out an itemized expense report accordingly. The user need only validate the report and hit submit, turning hours of expensing into sheer seconds.

NGCIO OneDrive Automation

Automating OneDrive account file methods

Each year, the Next Generation CIO Academy takes place at Deloitte University, training 100 soon-to-be CIOs for their roles. This kind of preparation requires 100 separate OneDrive accounts, so that participants can refer to reference materials over the course of the week. It would be a pain for the academy organizers to manually log into each provisioned OneDrive account and upload the needed files. Furthermore, should any files need to be deleted, moved, or shared, someone would have to again log into each account one by one. Given the broken skeleton of an existing tool that uploaded documents to multiple OneDrive accounts simultaneously, I rebuilt the tool from the ground up, ensuring that the existing functionalities of uploading files and deleting files actually worked. In addition to this, I added functionality to move all academy materials to one folder and automatically share that folder with each participant.

Jira to ADO Work Item Migration

Automating the migration of work items from Jira to Azure DevOps

The mobile channel at Deloitte started moving from Jira, an issue-tracking system, to the more holistic Azure DevOps. However, no tool existed to migrate issues automatically, so users had to manually create new issues in ADO, pull up the old Jira issues side by side, and re-populate all the fields themselves. To improve on this approach, I built a tool to automate the migration process. For each Jira issue, the tool fetched all the completed fields (e.g. type, status, priority, description), created a new issue in Azure DevOps, and then copied the old fields into the new issue. All that was needed on the user's end is to log into both Jira and VSTS, and then this console application takes care of the rest within minutes.

Microsoft Intune Exploration

Pushing provisioning profiles to apps nearing expiry using Intune

Provisioning profiles are certificates that are required for iOS apps to run on a device. Every year, provisioning profiles expire and must be renewed. Once they are renewed, if a user does not open an app for a 3-month period, the app will crash for the user, forcing them to reinstall. The goal of this research was to verify whether pushing a provisioning profile can be done through Intune, and if so, if it's possible to push them silently (i.e. with little disturbance to the user). We tested multiple scenarios and ultimately found that Intune is a great tool for pushing provisioning profiles to an app, but only if the app is installed after the profile is pushed.
